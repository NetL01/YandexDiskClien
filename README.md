# Советы по использованию функций, работащих с Яндекс.Диском
# Пример правильной работы: YandexDisk_Client.py upload D:/python/FileManagerProgect.py /Projects/FileManagerProgect.py
# Примечание: Файлы могут не иметь расширений, поэтому /test-dir можно интерпретировать как файл test-dir в корневом каталоге.

# Функции программы:
# Загрузить файл: upload путь в системе путь куда залить на диске
# Пример: YandexDisk_Clint.py upload D:/python/123.py /some/path/123.py
# Скачать файл: download Путь на Я.Диске  Путь у вас
# Пример: YandexDisk_Clint.py download /some/path/ing.jpg D:/images/ing.jpg
# Создать директорию: mkdir путь
# Переместить обьект: move откуда  куда *Только на Я.Диске*
# Список содержимого по пути: listdir путь
# Папка ли это is_dir путь
# Файл ли это: is_file путь
# Существует ли путь: exists путь
# Получить информацию о диске: get_disk_info
# Получить список ВСЕХ файлов: get_files
# Список последних загруженных файлов: get_last_uploaded
# Получить метаданные по пути: get_meta путь
# Получить список всех публичных ресурсов: get_public_resourses
# Получить метаданные корзины: get_trash_meta путь
# Получить тип данного корщины: get_trash_type
# Очистить корзину: remove_trash

# Примечания: параметры *kwargs не для начинающих пользователей, в большинстве функционала метода и без них должно # # хватать




# Шифрование:
# Выполнять операции с шифрованием рекомендовано в директории с установленной программой, вы можете вставить в параметры пути, но это не рекомендовано из-за частого отсутствия прав и многих других ситуаций.

# Как работает шифрование?
# У вас есть какой-либо файл, допустим с расширением .txt
# Вы прописываете в консоль: YandexDisk_Client encripto "название файла для шифрования" "название после процесса шифрования"
# Примечания: параметры без ковычек, название после шифрование рекомендовано называть полным названием файла до шифрования + .aes
# то есть data.txt -> data.txt.aes
# Если вы не захотите ввести пароль - он будет сгенерирован для вас самостоятельно
# Если вы забудите пароль - вы можете считать, что данные потеряны

# Как работает дешифрование?
# У вас есть файл, допустим тот же data.txt.aes
# Вы прописываете в консоль: YandexDisk_Client decripto "файл для дешифровки" "название после дешифровки"
# Примечание: всё без ковычек, в названии после дешифровки можно поставить любое значение, кроме формата и преждевременного откидывания формата .aes
# то есть data.txt.aes -> data.txt  иначе data.txt.aes -> some_name.txt
